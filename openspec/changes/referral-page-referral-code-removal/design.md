## Context

ReferralPage уже содержит рабочую реферальную механику. Задача — упростить интерфейс, убрав лишнее поле и сохранив основное действие: распространение ссылки.

## Goals / Non-Goals

**Goals:**
- Убрать поле referral_code полностью.
- Оставить реферальную ссылку и кнопку share.
- Сохранить аккуратный layout после удаления поля.

**Non-Goals:**
- Менять backend-логику referral link.
- Пересматривать всю страницу профиля.
- Менять бизнес-логику реферальной программы.

## Decisions

1. **UI упрощается до ссылки и действия share.**
   - Это уменьшает когнитивную нагрузку.
   - Альтернатива: оставить код рядом со ссылкой. Отклонено как лишнее.

2. **Spacing после удаления поля корректируется на уровне страницы.**
   - Необходимо сохранить визуальный баланс.
   - Альтернатива: просто скрыть поле без компенсации. Отклонено из-за возможных дыр в layout.

3. **Любые тексты и лейблы должны остаться понятными без отображения кода.**
   - Пользователь должен быстро понять, что именно можно поделиться.

## Risks / Trade-offs

- [Удаление поля может сместить блоки страницы] → проверить spacing и snapshot.
- [Старые тесты могут ожидать поле referral_code] → обновить ожидания.

## Migration Plan

1. Удалить поле из ReferralPage.
2. Проверить визуальный результат.
3. Обновить/добавить тесты на simplified layout.
4. Передать результат в devops/review flow.
